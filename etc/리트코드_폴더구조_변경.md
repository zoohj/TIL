## 리트허브 커스텀

### 1. 폴더 구조 변경의 핵심: `getPath` 함수
폴더를 `LeetCode/난이도/문제명/` 구조로 바꾸는 핵심 로직.
```javascript
const getPath = (difficulty, problem, filename) => {
  // 난이도 정보가 없으면 'Unrated'로 보내는 방어 로직
  const diff = typeof difficulty === 'string' && difficulty.length > 0 ? difficulty : 'Unrated';
  
  if (filename) {
    return `LeetCode/${diff}/${problem}/${filename}`;
  }
  return `LeetCode/${diff}/${problem}`;
};
```

### 2. 데이터 인자 순서 유의
함수를 호출할 때 인자 순서가 틀리면 `undefined` 폴더가 생김.
* **`getGitHubFile`**: `(token, hook, 폴더명, 파일명, 난이도)`
* **`upload`**: `(token, hook, 내용, 폴더명, 파일명, sha, 메시지, 난이도)`
* **통계(`stats.json`)**: 난이도 자리에 `Stats`를 넣어 별도 관리.

## 나중에 다시 세팅하는 법

1.  **클론 및 설치**:
    ```bash
    git clone [LeetHub 저장소 주소]
    cd LeetHub-2.0
    npm install
    ```
2.  **파일 교체**: 백업해둔 `leetcode.js`를 `scripts/` 폴더에 덮어쓰기 합니다.
3.  **빌드**:
    ```bash
    npm run build
    ```
4.  **확장 프로그램 등록**:
    * 크롬 주소창에 `chrome://extensions/` 입력.
    * **개발자 모드** ON.
    * **압축해제된 확장 프로그램을 로드** 클릭 후, 빌드된 **`dist`** 폴더 선택.
5.  **Stats 초기화**: 확장 프로그램 아이콘 클릭 후 **Clear Stats**를 눌러야 기존 문제들도 새 경로에 README가 올라감.

## ⚠️ 주의사항
* **수정 후엔 무조건 빌드**: 코드를 고친 뒤 터미널에서 `npm run build`를 안 하면 크롬엔 반영되지 않음.
* **새로고침**: 빌드 후엔 **1. 확장 프로그램 ↻**, **2. 리트코드 페이지 ⌘+R**을 꼭 해줘야 함.
* **`dist` 폴더의 위치**: 크롬에 등록한 `dist` 폴더를 다른 곳으로 옮기면 확장 프로그램이 깨짐. 옮긴 후엔 크롬에서 다시 경로를 잡아줘야 함.
