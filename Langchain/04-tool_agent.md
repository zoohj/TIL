**SQL 인젝션(SQL Injection)**은 쉽게 말해 **"DB에 보내는 질문지(쿼리)에 몰래 나쁜 명령어를 끼워 넣어, DB를 조종하는 해킹 수법"**입니다.

사용자가 입력하는 칸(로그인창, 검색창 등)에 정상적인 데이터 대신 SQL 문법을 적어서, 개발자가 의도하지 않은 동작을 하게 만드는 것이죠.

1. 어떻게 공격하나요? (예시: 로그인)
보통 로그인 로직은 DB에 이렇게 물어봅니다.

"사용자 아이디가 'admin'이고 비밀번호가 '1234'인 사람이 있어?"

🛑 정상적인 상황
입력: 아이디(admin), 비밀번호(1234)

SQL: SELECT * FROM users WHERE id = 'admin' AND pw = '1234'

결과: 일치하는 정보가 있으면 로그인 성공!

😈 SQL 인젝션 공격 (아이디 칸에 특수 문구 입력)
공격자가 비밀번호는 모르지만, 아이디 칸에 admin' -- 라고 입력한다면 어떻게 될까요?

입력: 아이디(admin' --), 비밀번호(아무거나)

SQL: SELECT * FROM users WHERE id = 'admin' --' AND pw = '아무거나'

여기서 핵심! SQL에서 --는 주석(무시하기) 처리를 뜻합니다.
즉, DB는 뒷부분인 AND pw = '...'를 아예 읽지 않고, 아이디만 맞으면 비밀번호 검사 없이 통과시켜 버립니다. 비밀번호 없이 관리자 계정으로 접속하게 되는 것이죠.





지금까지 살펴본 내용을 바탕으로, **LangChain을 이용한 지능형 에이전트 구축의 핵심 3단계**를 일목요연하게 정리해 드릴게요.

---

## 🏗️ 1단계: 도구(Tool) 설계의 정석
에이전트의 '팔과 다리'가 되는 도구들을 정의하는 단계입니다.

* **기능별 분리**: 계산(`multiply`), 외부 정보(`search_weather`), 내부 DB(`search_products`), 정책 정보(`get_policy`) 등 역할을 명확히 나눕니다.
* **보안(SQL Injection 방지)**: 사용자의 입력을 직접 쿼리에 넣지 않고 **파라미터 바인딩**(`:category`)을 사용하여 DB를 보호합니다.
* **레퍼런스 관리**: 모든 정보를 프롬프트에 넣는 대신, 목록만 보여주고 필요할 때만 도구로 호출(`get_policy`)하여 **토큰 비용을 절감**하고 정확도를 높입니다.



---

## 🧠 2단계: 퓨샷(Few-shot)을 통한 '지능' 주입
단순한 명령어보다 강력한 것은 **'예시'**입니다.

* **패턴 학습**: "A를 물어보면 B 도구를 쓰고 C로 답해"라는 흐름을 `Messages` 형태로 미리 제공합니다.
* **멀티 스텝(Multi-step)**: "가장 싼 거 3개 사면 얼마야?"처럼 **조회 → 계산**으로 이어지는 복잡한 논리 구조를 LLM이 스스로 파악하게 만듭니다.
* **일관성**: 에이전트가 추측하지 않고 반드시 도구를 사용하도록 가이드라인을 고정합니다.

---

## 🔄 3단계: 실행 루프 (The Agent Loop)
사용자의 질문이 끝날 때까지 반복되는 에이전트의 사고 과정입니다.

1.  **질문 수신**: `SystemMessage` + `Few-shot` + `HumanMessage`를 합쳐 LLM에게 전달.
2.  **판단(Reasoning)**: LLM이 "답변을 바로 할까? 도구를 쓸까?" 결정.
3.  **실행(Acting)**: 도구가 필요하면 코드를 실행하고 결과를 `ToolMessage`에 담아 다시 전달.
4.  **종료**: 더 이상 필요한 도구가 없을 때 사용자에게 최종 답변 제공.
