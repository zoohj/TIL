## sync / async SQLAlchemy URL 자동 파생 구조 설계
### 문제
Alembic 및 특정 동기 작업 시 postgresql+asyncpg:// 드라이버를 그대로 사용하면 MissingGreenlet 오류 발생.

그렇다고 .env에 두 개의 URL을 관리하는 것은 중복 설정이며, 휴먼 에러(비밀번호 변경 누락 등)의 원인이 됨.

### 해결: 단일 URL 기반 자동 드라이버 매핑
.env에는 하나의 DATABASE_URL만 정의하고, 코드 내부에서 이를 활용해 동기용(Sync)과 비동기용(Async) URL을 자동으로 생성하도록 구성함.

```Python
# 1. 원본 URL 로드
DATABASE_URL = settings.database_url

# 2. 어떤 형식이 들어와도 호환 가능하도록 자동 치환 logic 적용
# 비동기 드라이버가 포함되어 있으면 제거하여 Sync용 생성
SYNC_DATABASE_URL = DATABASE_URL.replace("postgresql+asyncpg://", "postgresql://")

# 동기 드라이버 형식이면 asyncpg를 붙여서 Async용 생성
ASYNC_DATABASE_URL = DATABASE_URL.replace("postgresql://", "postgresql+asyncpg://")

# 3. 각각의 목적에 맞게 엔진 할당
engine = create_engine(
    SYNC_DATABASE_URL,
    echo=settings.SQL_ECHO,
    pool_pre_ping=True,
)
```

### 배운 점 (Insight)
설정의 단일화 (SSOT): 개발자는 DATABASE_URL 하나만 신경 쓰면 되므로 유지보수가 매우 편리해짐.

유연한 대응: 사용자가 환경 변수에 postgresql://로 넣든 postgresql+asyncpg://로 넣든, replace 로직을 통해 양쪽 모두 대응 가능한 엔진을 생성할 수 있음.

Alembic과의 정합성: env.py에서 SYNC_DATABASE_URL을 참조하게 함으로써, 마이그레이션 도구(동기)와 FastAPI 애플리케이션(비동기) 간의 드라이버 충돌 문제를 근본적으로 해결함.

# 1. FastAPI 라우터 구조 설계 혼란

## 문제

초기에:

```text
routers/
  api.py
  endpoints/
```

구조를 어떻게 가져가야 하는지 혼란이 있었음.

특히:

* `api_router`
* `endpoints`
* `/api/v1`
* router 파일명
* include_router 위치

등이 뒤섞이며 import 오류 발생.

```python
ImportError: cannot import name 'api_router'
```

---

## 원인

* FastAPI의 Router Aggregation 패턴 이해 부족
* router와 endpoint의 역할이 섞여 있었음
* 파일명과 객체명이 충돌

---

## 해결

구조를 다음처럼 명확히 분리:

```text
routers/
├── api.py
└── endpoints/
    ├── youtube.py
    └── webhook.py
```

```python
# api.py
api_router.include_router(youtube.router)
```

---

## 회고 포인트

* FastAPI는 “router 조립 계층”을 따로 두는 게 유지보수에 유리
* `/api/v1` prefix는 초기에 미리 잡는 게 이후 breaking change를 줄임
