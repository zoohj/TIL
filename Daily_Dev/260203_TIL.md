### 1️⃣ BOJ 11724 - 연결 요소의 개수
--- 
#### 문제 요약

* **무방향 그래프**
* 정점 N개, 간선 M개
* 서로 연결된 정점 묶음(연결 요소)의 개수를 구하는 문제


#### 핵심 아이디어

* **DFS / BFS로 그래프 탐색**
* "DFS를 시작한 횟수 = 연결 요소 개수"


#### 사고 흐름

1. 그래프를 **인접 리스트**로 표현
2. 방문 여부를 저장할 `visited` 배열 사용
3. 1번 정점부터 N번 정점까지 순회
4. 아직 방문하지 않은 정점을 만나면

   * 해당 정점에서 DFS 시작
   * 연결된 모든 정점 방문 처리
   * 연결 요소 개수 +1


#### 인접 리스트 초기화

```python
adj = [[] for _ in range(n + 1)]
```


#### DFS 핵심 개념

* DFS 시작 시 **자기 자신부터 방문 처리**
* 인접한 정점 중 방문하지 않은 정점으로 재귀 이동


#### 주의할 점

* 파이썬 재귀 DFS 사용 시 **재귀 한도 초과 가능**
* 해결 방법:

```python
import sys
sys.setrecursionlimit(10**7)
```

### 2️⃣ BOJ 1966 - 프린터 큐
---

#### 문제 요약

* 문서마다 중요도(1~9)
* 큐의 맨 앞 문서보다 **중요도가 높은 문서가 하나라도 있으면** 뒤로 이동
* 아니면 출력
* 특정 문서가 **몇 번째로 출력되는지** 구하기


#### 핵심 질문

> "지금 큐 맨 앞 문서보다 중요한 문서가 큐 안에 있나?"

* YES → 뒤로 보냄
* NO → 출력


#### 큐에 저장할 정보

* **문서 위치(index)**
* **중요도(priority)**

```python
(index, priority)
```

#### deque 활용 포인트

* 맨 앞 확인: `dq[0]`
* 맨 앞을 뒤로 보내기:

```python
dq.append(dq.popleft())
```

* 원형 큐 동작: `rotate()`


#### 처음 구현이 복잡해진 이유

* 중요도 개수 배열
* 중요도별 중첩 반복
* 미리 최적화하려다 구조가 복잡해짐

이 문제는 **입력 크기가 작아서 단순 시뮬레이션이 정답**

---

#### 클린 코드로 가는 사고법

1. 문제를 **한 문장으로 요약**

   * "큐 맨 앞을 계속 검사하다가, 출력되면 카운트"
2. 상태만 저장 (index, priority)
3. **while 하나로 시뮬레이션**

---

#### 교훈

* 시뮬레이션 문제는

  > "사람이 손으로 따라가는 흐름 그대로 코드로 옮긴다"
* 처음부터 최적화 ❌
* 단순한 구조부터 만든다 ⭕
