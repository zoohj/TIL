### 문제 복기

### 1. 알고리즘 조합 검증 (배열)

* **문제 설명**: 사용 가능한 알고리즘 리스트가 주어지고, 테스트 케이스로 '배열 안의 배열' 형태의 데이터가 제공됨.
* **미션**: 각 테스트 케이스(배열)에 포함된 알고리즘들이 모두 '사용 가능 알고리즘'에 속하는 경우의 수를 구하기.
* **핵심 로직**:
* 허용 리스트에 없는 알고리즘이 하나라도 포함되면 카운트 제외.
* `Set` 자료구조를 사용하여 포함 여부($O(1)$)를 빠르게 체크하는 것이 관건.



---

### 2. 리뷰어 친목 점수 계산 (구현/시뮬레이션)

* **데이터 구성**: 리뷰어 ID, 팀 ID, 부여 점수, 소속 팀 정보.
* **계산 공식**: $\text{친목 점수} = \frac{\text{내부 팀 평가 평균}}{\text{외부 팀 평가 평균}}$
* **예외 처리 (Edge Case)**:
* **내부 팀** 평가 횟수가 0인 경우 → `lowest` 반환.
* **외부 팀** 평가 횟수가 0인 경우 → `highest` 반환.


* **학습 포인트**: 소속 팀 데이터를 해시 맵(Hash Map)으로 관리하여 `ID`별 `Team` 매칭을 효율화하고, 분모가 0이 되는 상황에 대한 조건문 처리가 핵심.

### 3. 엘리베이터 호출 최적화 (시뮬레이션)

* **문제 설명**: 특정 층에서 호출이 발생했을 때, 여러 대의 엘리베이터 중 어떤 엘리베이터가 응답하며 현재 어디에 위치하게 되는지 계산.
* **핵심 로직**: 현재 엘리베이터의 층수, 이동 방향, 대기 리스트 등을 고려한 거리 계산 알고리즘 구현.

### 5. SQL: 몬스터 진화 및 병합 데이터 추출

* **테이블 구조**:
* `Monster`: `id`, `name`, `power`
* `Evolution`: `id`, `name`, `monster1_id`, `monster2_id`


* **요구 사항**: 기본 몬스터와 진화 몬스터의 정보를 통합하여 출력.
* **출력 컬럼**: `아이디`, `이름`, `전투력`, `병합여부(0 또는 1)`
* **해결 전략**:
* `Monster` 테이블에서 기본 정보 추출 (병합여부 0).
* `Evolution`과 `Monster`를 2번 JOIN하여 `monster1`과 `monster2`의 전투력 합산 (병합여부 1).
* 두 결과 집합을 `UNION ALL`로 결합.
