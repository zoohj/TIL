## 🎧 간단 정리

### 파이썬 기초 개념

### 1. 탐색과 효율성 (Searching)

* 🫥 `#1920 수 찾기`
* **핵심 개념:** **이분 탐색 (Binary Search)**
* **문제점:** 리스트에서 `in` 연산을 사용하면 $O(N)$의 시간이 걸려 데이터가 많을 때(10만 건 이상) 시간 초과 발생.
* **해결:** 데이터를 먼저 정렬()한 뒤, 범위를 절반씩 좁혀가는 이분 탐색()을 사용하거나, 해시 기반의 `set` 자료형()을 활용함.



### 2. 선형 자료구조 (Deque)

* 🫥 `#2161 카드1`, `#1158 요세푸스 문제`, `#13335 트럭`
* **핵심 개념:** **FIFO (First-In-First-Out) & 시뮬레이션**
* **핵심 도구:** `collections.deque`
* **요세푸스:** `rotate()`를 사용해 '앞사람을 뒤로 보내는' 원형 구조를 효율적으로 구현 ( 연산). 음수를 넣으면 반대로 이동.
* **트럭:** 다리를 '고정된 길이의 큐'로 모델링. 1초마다 `popleft()`로 전진시키고, 무게 조건에 따라 `append(트럭)` 혹은 `append(0)`을 결정하는 시뮬레이션 논리가 중요함.



### 3. 후입선출 자료구조 (Stack)

* 🫥 `#9012 괄호`, `#4949 균형잡힌 세상`
* **핵심 개념:** **짝 맞추기 (LIFO - Last-In-First-Out) & 매핑(Mapping) 활용**
* **핵심 원리:** 
  * **LIFO**: '가장 나중에 열린 괄호가 가장 먼저 닫혀야 한다'는 규칙을 Deque으로 구현.
  * **매핑 활용**: if-elif를 나열하는 대신 **딕셔너리(braket_dic)**를 사용해 닫는 괄호와 여는 괄호의 쌍을 관리함.
    * 코드의 가독성이 좋아지고, 유지보수가 쉬움. 
       ```python
        braket_dic = {
          ')' : '(',
          '}' : '{',
          ']' : '[',
          '>' : '<'
      }
      open_braket = set(braket_dic.values())
      close_braket = braket_dic.keys()
      ```
* **주의점:** 
  * 닫는 괄호가 나왔을 때 Deque이 비어있는지 확인 (`IndexError` 방지).
  * 모든 검사가 끝난 후 Deque이 완전히 비어있는지 확인. => 결과값 출력
  * `[(])`와 같은 교차 구조는 Deque으로 해결 불가능.



### 4. 정수론 (Mathematics)

* 🫥 `#1929 소수 구하기`
* **핵심 개념:** **에라토스테네스의 체, 인덱스 범위 줄이기(제곱근)**
* **효율성:** 숫자를 하나씩 소수인지 판별하는 것()보다, 소수의 배수를 지워나가는 '체' 방식이 대량의 소수를 구할 때 훨씬 빠름.
* 무의미한 반복을 줄이기 위해 인덱스를 줄여나가는 것이 중요함!
* **구현 팁:** `[True] * (n + 1)` 형태의 상태 리스트를 만들어 인덱스로 접근하고, `range(start, stop, step)`를 활용해 배수를 효율적으로 제거함.
