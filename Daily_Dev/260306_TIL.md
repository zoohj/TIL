## React 기초 및 컴포넌트 설계

### 1. JSX 문법의 규칙

* **엄격한 태그 닫기**: `<img>`, `<br>`, `<li>` 등 모든 태그는 반드시 `/>` 또는 닫는 태그로 끝내야 함.
* **Property 명칭**: HTML의 `class` 대신 `className`을 사용해야 함.
* **카멜 케이스(camelCase)**: `stroke-width` 대신 `strokeWidth`처럼 속성명을 작성함.

### 2. 컴포넌트와 변수 위치

* **함수 밖 변수**: 컴포넌트가 리렌더링되어도 다시 생성되지 않아 성능상 유리함. 변하지 않는 고정 데이터(메뉴판, 설정값 등)를 둘 때 최적.
* **함수 안 변수**: 컴포넌트의 상태(`state`)나 전달받은 값(`props`)에 따라 변하는 데이터를 처리할 때 사용.

### 3. 컴포넌트 계층 구조 (Composition)

* **부모-자식 관계**: 큰 단위의 컴포넌트(`MenuBoard`) 안에 작은 단위의 컴포넌트(`MenuTitle`, `MenuList`)를 넣어 조립함.
* **데이터 전달(Props)**: 부모가 자식에게 데이터를 물려줄 수 있음.

### 4. Props 활용 및 주의사항

* **객체 Props**: 여러 데이터를 하나로 묶어 `{ book }` 처럼 객체 형태로 전달하면 관리가 편리함.
* **예약어 피하기**: `class`는 자바스크립트 자체의 예약어이므로, Props 이름으로 사용하면 **500 에러**나 문법 오류가 발생함. (대신 `classNum`, `group` 등 사용 권장)

### 5. 리스트 렌더링

* **`.map()` 사용**: 배열 데이터를 반복해서 화면에 그릴 때 필수.
* **Key 값**: 각 항목에는 고유한 `key` 속성을 부여해야 리액트가 효율적으로 변화를 감지함.
