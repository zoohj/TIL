# 스프링 빈 등록과 의존관계 설정

> **Date:** 2026-03-07 <br>
> **Topic:** Spring Bean, 의존성 주입(DI), 컴포넌트 스캔, 자바 설정 파일

---

## 1. 의존성 주입 (Dependency Injection, DI)

객체 간의 의존관계를 외부(스프링 컨테이너)에서 넣어주는 것을 말함.

* **동작 원리:** 생성자에 `@Autowired`가 있으면 스프링이 연관된 객체를 컨테이너에서 찾아 주입.


* **주입 방법 3가지:** 필드 주입, setter 주입, **생성자 주입**.


* **권장 방식:** 실행 중에 의존관계가 동적으로 변하는 경우는 거의 없으므로 **생성자 주입**을 권장.


* *참고:* 생성자가 1개만 있으면 `@Autowired`를 생략할 수 있음.



---

## 2. 스프링 빈을 등록하는 2가지 방법

### ① 컴포넌트 스캔과 자동 의존관계 설정

* **원리:** `@Component` 애노테이션이 있으면 스프링 빈으로 자동 등록.


* **대상:** `@Controller`, `@Service`, `@Repository`는 내부에 `@Component`를 포함하고 있어 스캔 대상이 됨.


* **특징:** 실무에서 주로 정형화된 컨트롤러, 서비스, 리포지토리에 사용.



### ② 자바 코드로 직접 스프링 빈 등록하기

* **방법:** `@Configuration`과 `@Bean` 애노테이션을 사용하여 직접 설정 클래스를 작성.


* **장점:** 정형화되지 않거나, **상황에 따라 구현 클래스를 변경해야 할 때** 설정 파일만 수정하면 됨.


* 예: 향후 메모리 리포지토리를 다른 리포지토리로 변경할 계획이 있다면 이 방식이 유리.


---

## 3. 핵심 체크포인트 💡

* **싱글톤(Singleton) 원칙:** 스프링은 빈을 등록할 때 기본적으로 **유일하게 하나만 등록하여 공유**. 따라서 같은 빈이면 모두 같은 인스턴스.


* **DI의 전제 조건:** `@Autowired`를 통한 의존성 주입은 **스프링이 관리하는 객체(스프링 빈)** 사이에서만 동작.


* 내가 직접 `new`로 생성한 객체에서는 `@Autowired`가 작동하지 않음.


* **오류 해결:** `Consider defining a bean of type...` 오류가 발생한다면, 해당 클래스가 스프링 빈으로 등록되지 않았는지 확인.
